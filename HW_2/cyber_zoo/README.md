## Инструкция по запуску

1. Установить зависимости:
```bash
npm install
```

2. Запустить приложение:
```bash
npm start
```

3. Открыть в браузере: http://localhost:3000

## Реализованный функционал

### Хуки React
- `useState` + `useEffect` для загрузки данных и управления состоянием
- `useRef` для DOM-эффектов аватара
- `useReducer` в карточке питомца для управления состоянием
- `useMemo` для мемоизации отфильтрованного списка и списка видов
- `useCallback` для мемоизации колбэков
- `React.memo` для оптимизации PetCard

### Кастомные хуки
1. `useEventLog` - работа с глобальным логом событий
2. `usePetLifecycle` - автоматическое уменьшение энергии и обновление настроения

### Функциональность
- Загрузка питомцев из pets.json с задержкой
- Skeleton Loader во время загрузки
- Фильтрация по виду питомца
- Действия питомца: Покормить, Уровень +, Приласкать, Сброс
- Таймер автоматического уменьшения энергии (каждые 10 секунд)
- Глобальный лог событий через useContext

### Стилизация
1. **SCSS** (global.scss) - базовая сетка приложения
2. **module.scss** (PetCard.module.scss) - стили карточки питомца
3. **inline styles** - динамический цвет индикатора настроения
4. **styled-components** (ActionButton.styled.ts) - кнопки взаимодействия
5. **Material UI** - Select для фильтрации, Drawer для лога, Skeleton Loader

## Доработки на оценку 9

### Сохранение состояния в localStorage

Реализовано сохранение данных в localStorage:

#### 1. Сохранение состояния питомцев
- Состояние каждого питомца (энергия, уровень, настроение) сохраняется в localStorage
- Используется ленивая инициализация в `useReducer` для загрузки сохраненных данных
- Ключ хранения: `cyber_zoo_pet_{id}`

#### 2. Сохранение лога событий
- История всех событий сохраняется в localStorage
- Очистка лога также очищает localStorage
- Ключ хранения: `cyber_zoo_events`
